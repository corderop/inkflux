# Tools

* Use context7 tools to access to the updated documentation of any tool/framework if needed.

# General guidelines

* Do not submit a lot of code for user review. Split big changes in smaller ones so the user can easily review them.

## Tech stack
* Typescript
* Astro
* Vitest
* pnpm 

# Contribution Guidelines

* Use Conventional Commits standard for commits
* Always use typescript.

# Testing

* Use vitest for testing.
* Do not test Frontend components yet.
* Test library and utilities.
* When you are implementing a testable code (libraries, utilities), implement first the tests for it and then the production code. Follow a TDD approach per phases.
